App Summary

Retail Demand Forecasting System

Research-oriented Python pipeline for retail demand forecasting and inventory decision evaluation.

What It Is

This repo is a research-oriented forecasting pipeline for retail inventory decisions under uncertainty, stockouts, and drift. It uses the FreshRetailNet-50K dataset by default and evaluates models with predictive, probabilistic, and economic metrics.

Who It's For

Primary user: a data scientist, operations analyst, or student researcher who wants to test demand forecasts against inventory costs in a reproducible experiment pipeline.

What It Does

  • Loads FreshRetailNet-50K from Hugging Face with local raw-cache and processed-panel caching.
  • Cleans a daily store-product panel and filters to configurable series with enough history.
  • Builds leakage-aware lag, rolling, calendar, discount, stockout, weather, and static-ID features.
  • Compares a seasonal naive baseline with a global boosting model plus quantile forecasts.
  • Runs walk-forward backtesting with expanding training windows and optional retraining each fold.
  • Converts forecasts into order quantities with a single-period newsvendor policy.
  • Writes Markdown reports, CSV metrics, prediction outputs, and optional plots per run.

How It Works

CLIretail_forecasting.run parses args, loads YAML settings, and starts run_experiment.
Datadata/fresh_retailnet.py reads the train split from Hugging Face or local cache, then writes a processed parquet panel.
Featuresfeatures/engineering.py builds the supervised frame and lead-time demand target from historical lags and rollups.
Forecastingforecasting/backtesting.py builds walk-forward folds; pipeline.py scores SeasonalNaiveModel and AutoBoostingModel.
Decisioninventory/newsvendor.py turns point or quantile forecasts into order quantities and computes overstock/stockout cost.
Outputsevaluation/reporting.py writes report.md, CSV summaries, predictions, and optional plots under reports/.
External service in repo evidence: Hugging Face dataset storage. Web app, API server, and deployed service: Not found in repo.

How To Run

  1. Install uv if needed: brew install uv
  2. Create and activate a virtualenv: uv venv then source .venv/bin/activate
  3. Install dependencies: uv sync --extra dev
  4. Run the default experiment: uv run python -m retail_forecasting.run --config configs/default.yaml
Optional accelerated backends: uv sync --extra dev --extra ml. Run outputs are written to a timestamped folder in reports/.
